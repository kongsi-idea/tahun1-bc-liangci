# 量词大冒险 — 进度记录

一年级华文「5.3 认识各类词语——量词」的自主学习 + 闯关游戏。改编自 `../../mario-game/index.html`
（Phaser 3 像素闯关引擎），题库换成 17 个量词，前面加了强制学习 slides + 分数达标闸门。

## 怎么玩（本地试玩）

```
cd teaching-tools/tahun1-bc-liangci
python3 -m http.server 8744
```
浏览器打开 http://localhost:8744/ 。

## 整体流程
1. **选人**：选班级（1I / 1G，沿用《一年级华文闯关问答》同一份名单）→ 点自己的名字
2. **Slides 学习**（19 页：1 页量词概念介绍 + 17 个量词 + 1 页总结）：
   - 门槛不分「第一次学习」和「答错重学」，都一样：把 17 个量词、102 个例子短语（如「一座大山」）**逐一点击确认读过**，全部点完「开始闯关游戏」才解锁——没有时间限制，因为点完 102 个已经需要一定时间，不需要额外计时
   - 每页至少停留 0.8 秒才能翻下一页（防止狂点跳过）
3. **游戏闯关**：Phaser 版 Mario，一轮固定 **10 题**（题库 119 题随机抽 10，不重复），沿用原本双段跳/坑洞/终点旗杆机制，另外加了：
   - 地面怪物（青蛙，巡逻+可踩）+ **天上的怪兽**（老鹰，飞行巡逻+轻微上下浮动，可踩可撞）
   - 每个问答站除了原本的问答方块+砖块，额外叠了一个**比较矮的小箱子**在砖块正上方（纯装饰+可踩）
4. **结算**：正确率 ≥ 70%（`PASS_RATE` 常量可调）算达标
   - 达标：可以「再玩一次」（新一轮 10 题）、「重新看资料」、看排行榜，或换人
   - 不达标：只能「重新学习资料」，强制回到 slides 重新点完全部 102 个例子，没有跳过游戏的路径
5. **排行榜**：历史最高分 + 累计挑战次数，阶段一先用 localStorage 模拟，接了 Google Sheet 后端后自动变成全班共享

## 文件结构
- `index.html` — 全部前端代码（选人/学习/游戏/结算/排行榜五个画面 + Phaser 场景）
- `Code.gs` — 排行榜后端（Google Apps Script，纯 JSON API，不含任何页面），部署方式见 `部署步骤.md`
- `assets/` — 直接复用 `mario-game/assets` 的 SunnyLand 美术素材（同一份文件，非任天堂角色，无版权顾虑）

## 关键设计决定
- **架构**：网页本体部署 Vercel/GitHub（静态站，跟 teaching-tools 其他工具一致）；共享排行榜用 Google Sheets 存资料，但只透过一个「只回传 JSON、学生看不到」的 Apps Script Web App 读写——用户明确要求不要整个网站放在 script.google.com 上面，只接受 Apps Script 当安静的 API 桥梁
- **学习门槛 = 点击确认，不计时**：一开始设计过「翻完 19 页 + 满 5 分钟」的计时门槛，用户反馈「点完 102 个卡片本来就要花不少时间了，不需要再计时」，改成不分场景统一用「102 个例子逐一点击确认」当门槛，答错重学时也是重新点一遍全部 102 个（`confirmedPhrases` 每次进入 slides 都会清空重来）
- **每轮 10 题**：`ROUND_SIZE` 常量控制，从最初的 20 题（用户嫌太长）调整为 10 题；关卡长度、怪物/宝石站点都跟着 `ROUND_SIZE` 自动生成
- **题库来源**：17 个量词、每个 6 个四字短语 + 1 个例句 = 119 题候选池，每轮 `shuffleArr().slice(0,ROUND_SIZE)` 随机抽（已用 node 脚本模拟验证：选项 4 个不重复、答案索引有效）
- **达标标准**：正确率门槛（用户选择），不是错题数上限
- **踩坑记录：宝石卡进箱子里**——最初每段关卡中间那颗装饰宝石的 x 坐标跟砖块几乎重叠（只差 2px），学生反馈"看起来像卡在箱子里"；修正为明确放在砖块正上方、留出清楚的垂直间距（`BLOCK_Y - TILE*1.4`），改用 `{x,y}` 物件取代原本靠 `i%3` 猜高低的写法，逻辑更直观也更不容易再犯同样的错
