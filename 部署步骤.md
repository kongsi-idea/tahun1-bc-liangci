# 部署步骤（阶段二：把共享排行榜接起来）

这个工具分两块：
1. **网页本体**（`index.html`）——部署在 Vercel/GitHub，学生打开的就是这个网址
2. **排行榜后端**（`Code.gs`）——部署成 Google Apps Script 网络应用，只回传 JSON，学生看不到它，纯粹是网页背后用来读写 Google Sheet 的桥梁

阶段一（现在）：`index.html` 里 `API_URL` 是空的，排行榜会自动退回浏览器本机 `localStorage` 模拟，可以先把整个学习+游戏流程测试完整。等确认没问题，再照下面步骤把共享排行榜接上。

## 第一步：新建 Apps Script 项目
1. 打开 https://script.google.com
2. 左上角「新建项目」，项目名称改成「量词大冒险 - 后端」

## 第二步：贴上后端代码
1. 左边默认文件 `Code.gs`，点开，删掉原本内容
2. 把这个资料夹的 `Code.gs` 全部内容复制贴上

## 第三步：保存并部署
1. `Cmd+S` 保存
2. 右上角「部署」→「新建部署」
3. 类型选择「网页应用」（没看到就先点齿轮图标选类型）
4. 设置：
   - **执行身份**：我（自己的账号）
   - **有权访问的用户**：所有人
5. 点击「部署」，跳出 Google 授权画面，选自己的账号，「继续」/「允许」
6. 出现"未经验证的应用"警告是正常的（自己写的小工具没送审），点「高级」→「转至……（不安全）」→「允许」
7. 部署完成会给一个网址，类似：
   `https://script.google.com/macros/s/xxxxxxxxxxxx/exec`

## 第四步：把网址填进网页
打开 `index.html`，找到这一行（在 `<script>` 里靠前的位置）：
```js
const API_URL = '';
```
把网址贴进去：
```js
const API_URL = 'https://script.google.com/macros/s/xxxxxxxxxxxx/exec';
```
保存。

## 第五步：部署网页本体到 Vercel/GitHub
跟这个资料夹底下其他工具（`tahun1-bc-zaoju`、`tahun1-bc-shizi` 等）流程一样：
1. 独立 Git 仓库（GitHub org: `kongsi-idea`，仓库名 = slug `tahun1-bc-liangci`）
2. 连到 Vercel，部署根目录就是这个资料夹
3. 部署完成后测试：换两台不同的设备/浏览器提交分数，确认两边都能在排行榜看到彼此的成绩

## 之后要改题目/改内容怎么办？
- 量词题库、slides 内容都在 `index.html` 里，直接改对应的 `WORDS` 数组或题库逻辑，然后重新部署 Vercel 即可，`Code.gs` 不需要动
- 如果要改排行榜的资料结构（比如加新栏位），才需要回到 `Code.gs`，改完后「部署」→「管理部署」→ 编辑（铅笔）→「新版本」→「部署」（网址不会变）
